## 1. 画面概要

この画面でユーザーが達成する目的を簡潔に記述する。

  

- 目的：ユーザーが達成した特定のタスク進捗を記録する

- 利用者：ユーザー

- 画面の主な役割：ユーザーが達成した特定のタスク進捗を記録しプロジェクト進捗バーに進捗を反映

## 2. ストーリー / ユースケース

画面に到達する前後の流れを文章または箇条書きで記述する。

- 到達パス（どこから遷移してくるか）：タスクを

- 完了条件：完了できる
